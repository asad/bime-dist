Interactive Examples

Try BIME features live in your browser. Each example includes a working mini-editor and the JavaScript code to reproduce it.

Try SMILES

Type or pick an example 
var editor = new MolEditor('container', '100%', '280px', {
    options: 'hydrogens'
});

// Load a SMILES string
editor.readGenericMolecularInput('c1ccccc1');

// Read back canonical SMILES
console.log(editor.smiles());

Try SMARTS

Substructure search
Target molecule: Aspirin (CC(=O)Oc1ccccc1C(=O)O)
// Load aspirin into the editor
editor.readGenericMolecularInput('CC(=O)Oc1ccccc1C(=O)O');

// Match a SMARTS pattern — matching atoms highlight in teal
var count = editor.matchSmarts('[OH]');
console.log('Matches found:', count);

// Programmatic matching (no editor needed)
var mol = new Molecule();
SmilesParser.parse('CC(=O)Oc1ccccc1C(=O)O', mol);

var query = new Molecule();
SmartsParser.parse('[OH]', query);

var mappings = SmartsMatch.match(mol, query);
console.log(mappings.length, 'match(es)');

Try Reactions

Reactant → Product 
// Enable reaction mode
var editor = new MolEditor('container', '100%', '280px', {
    options: 'hydrogens,reaction'
});

// Load reaction SMILES (>> separates reactants and products)
editor.readGenericMolecularInput('CC(=O)O.OCC>>CC(=O)OCC.O');

// Atom-mapped reaction
editor.readGenericMolecularInput('[CH3:1][OH:2]>>[CH2:1]=[O:2]');

Try Export

SVG & PNG download

Draw or edit the molecule below, then export it as a publication-quality image.

// SVG export
var svg = ImageExport.toSVG(editor.molecule, {
    width: 400, height: 300,
    padding: 40,
    background: '#ffffff',
    bondWidth: 2.5,
    fontSize: 16,
    showHydrogens: true
});
ImageExport.downloadSVG(svg, 'molecule.svg');

// PNG export at 4x resolution
ImageExport.toPNG(editor.molecule, { scale: 4 })
    .then(function(blob) {
        ImageExport.downloadPNG(blob, 'molecule.png');
    });

Try Validation

SMILES checker 
// Validate a SMILES string
var result = editor.validateSmiles('c1ccccc1');
// { valid: true, atoms: 6, bonds: 6, smiles: 'c1ccccc1' }

// Invalid: unclosed ring
var bad = editor.validateSmiles('C1CCC');
// { valid: false, error: 'Unclosed ring 1 ...' }

// Invalid: unknown atom
var bad2 = editor.validateSmiles('XYZ');
// { valid: false, error: 'Unknown atom symbol "X" ...' }

// Valence warning
var warn = editor.validateSmiles('C(C)(C)(C)(C)C');
// { valid: true, warnings: ['Valence violation ...'] }

5. Atom-Atom Mapping (AAM)

RDT (Reaction Decoder Tool)

Auto-map atoms across reactant and product using BIME's pure-JS RDT port. Click an example to load a reaction, then click Auto-map. Mapped atoms display the :N superscript across both sides.

How does this work?

BIME ships a faithful pure-JavaScript port of the published Reaction Decoder Tool (RDT) algorithm. Four pairing strategies (MIN, MAX, MIXTURE, RING) each compute a candidate atom mapping driven by maximum-common-substructure similarity between every (reactant, product) component pair. A fitness function scores each candidate by mapped-atom coverage, bond-change count, and ring-bond preservation; the lowest-scoring strategy wins. The whole pipeline runs offline in your browser with no server calls. See Rahman et al. RDT 2016 and EC-BLAST 2014 in the references below.

6. Maximum Common Substructure (MCS)

SMSDMCS engine

Find the largest shared substructure between two molecules. The MCS atoms highlight in teal in the left editor; the size, mapping, and Tanimoto score appear below.

How does this work?

The Small Molecule Subgraph Detector (SMSD) MCS engine builds molecular graphs, applies neighborhood-label-fingerprint screening, expands seed pairs into candidate cliques, and selects the largest connected (or disconnected) common substructure. Tanimoto similarity is computed from the MCS size relative to both inputs. The implementation runs entirely in your browser. See Rahman et al. SMSD 2009 in the references below.

7. Substructure Search (SUB) — SMARTS

VF2++ matcher

Find a sub-pattern (a SMARTS query) inside a target molecule. Matched atoms highlight in teal. Useful for substructure-based property prediction, scaffold detection, and functional-group analysis.

How does this work?

The query SMARTS string is parsed into a query molecule with per-atom and per-bond constraints (element, charge, ring membership, valence, recursive sub-patterns, etc.). A VF2-based subgraph isomorphism search then enumerates all mappings from the query atoms to target atoms that satisfy every constraint. All symmetry-equivalent mappings are returned. See Rahman et al. SMSD 2009 in the references below.

8. Bond Changes (BC)

RDT.annotateBondChanges

Visualise which bonds are formed, cleaved, or change order during a reaction. Bond-change annotation runs after AAM. Click an example, then Auto-map, then Show Bond Changes.

How does this work?

After AAM produces a reactant-to-product atom map, BIME walks every reactant bond: if either endpoint is unmapped (or maps to a product where no equivalent bond exists) the event is cleaved; if the bond order differs between sides the event is orderChange. Product bonds whose endpoints are images of mapped reactant atoms but had no reactant counterpart are formed. Stereo flips at mapped atoms become stereoChange events. See Rahman et al. RDT 2016 in the references below.

9. Canonical SMILES (Canon)

SmilesWriter.write

Convert any SMILES variant to its canonical form. BIME uses a Morgan-style canonical labelling that picks a unique atom-order so the same molecule always gives the same SMILES — essential for searching, deduplication, and database keys.

How does this work?

BIME computes a Morgan-style canonical atom ordering (initial invariants of atomic number, charge, isotope, aromaticity, and H-count; iterated by combining neighbour invariants with a deterministic hash; tie-broken by a stable atom-environment fingerprint), then performs a deterministic DFS from the highest-rank atom, emitting ring-closures and branches in canonical rank order. The result is invariant under input atom-order permutation, so two different SMILES strings of the same molecule converge to one canonical output. Stereo round-trip drift on polycyclic multi-stereocentre molecules is a known limitation — see tests/test_round_trip.js.
